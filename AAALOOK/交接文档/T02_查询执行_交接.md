## 2. 本题做了什么

本题完成了 RMDB 的元数据管理、建表删表、语义检查、查询计划生成和基础执行算子。系统现可执行 `create/drop/show`、`insert/delete/update/select`，支持单表条件过滤、投影、笛卡尔积和多表连接。非法表名、字段名、值数量和类型等情况会输出 `failure`。

## 3. 主要修改文件

| 文件路径 | 改动说明 |
|---|---|
| `src/system/sm_manager.cpp` | 实现数据库打开关闭、建表删表和元数据持久化 |
| `src/analyze/analyze.cpp` | 完成增删改查语义检查及常量类型处理 |

| `src/execution/executor_*.h` | 实现扫描、投影、连接、更新和删除算子 |
| `src/rmdb.cpp` | 语法错误时写入 `failure` |
| `src/query_execution_test.cpp` | 增加题目二端到端回归测试 |

## 4. 核心实现逻辑

1. Analyze 校验表、列和值类型，并把条件常量转换成记录格式。
2. Planner 按 SQL 表顺序创建扫描节点，将单表条件下推，并构造左深连接树。
3. SeqScan 逐条过滤记录；NestedLoopJoin 枚举左右输入；Projection 重排并复制目标列。
4. Portal 收集更新或删除目标 RID，再由对应执行算子修改记录文件。
5. SmManager 在建表、删表和关闭数据库时同步元数据与文件句柄。

## 5. 关键注意事项

- 浮点字段允许使用整数常量，例如 `score > 90` 和 `score = 0`，分析阶段会转换为浮点数。
- 当前题目不要求索引扫描、排序和索引维护；后续实现索引后需同步更新增删改算子。
- 修改查询或存储模块后，应同时运行题目二端到端测试和题目一存储回归测试。
