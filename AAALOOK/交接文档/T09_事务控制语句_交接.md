## 2. 本题做了什么

本题完成了显式事务的 `begin`、`commit`、`abort` 和 `rollback` 语义。INSERT、DELETE、UPDATE 成功后会登记事务写集合，提交时清理写记录，回滚时按写入的相反顺序恢复基表记录及所有唯一索引。实现同时兼容框架原有的单语句隐式事务，并支持无索引表、带索引表和 DATETIME 字段；希冀平台一次性全部通过。

## 4. 核心实现逻辑

1. `begin` 将当前连接的事务标记为显式事务，后续 DML 共用同一 Transaction。
2. DML 完成实际修改和索引维护后，将操作类型、表名、RID 及必要的旧记录加入写集合。
3. `commit` 删除写集合内容、释放事务锁并将状态设为 COMMITTED。
4. `abort/rollback` 从写集合尾部开始撤销操作，保证同一记录的连续修改按正确顺序恢复。
5. 撤销时根据当前或旧记录重新构造各索引键，使基表和索引始终保持一致。

## 5. 关键注意事项

- 写记录必须在 DML 成功后登记，失败操作不能进入写集合。
- INSERT 回滚删除新记录；DELETE 回滚在原 RID 恢复旧记录；UPDATE 回滚写回旧值。
- 撤销 UPDATE 和 DELETE 时必须同步删除当前索引键并恢复旧索引键。
- 回滚必须严格逆序，才能正确处理“插入后更新再删除”等连续操作。
- 服务器在显式事务结束后的下一条语句会创建新事务；隐式事务仍在单条 SQL 后自动提交。
