## 2. 本题做了什么

本题为 RMDB 实现了单列及多列唯一索引，支持创建、删除和 `show index from table`。查询规划器能够按照最左匹配原则选择联合索引，执行层使用 B+ 树完成单点与范围扫描。插入、删除和更新会同步维护基表上的所有索引，并在建索引及数据修改时检查唯一性。希冀平台一次性全部通过。

## 4. 核心实现逻辑

1. B+ 树内部节点保存各子树首键，叶节点按键排序并通过双向链表连接。
2. 插入时二分定位叶节点，检测重复键；节点满后平均分裂并递归更新父节点。
3. Planner 按索引列顺序匹配条件，支持等值前缀加一个范围字段，且不依赖 SQL 条件顺序。
4. IndexScan 使用各类型的最小/最大值补全联合键，计算精确的扫描起止位置，再用完整条件过滤结果。
5. DML 修改前先验证全部唯一约束，成功后同步修改记录文件和所有相关索引。

## 5. 关键注意事项

- 联合索引遵循最左匹配；缺少首列条件时不能直接使用后续列。
- `>`、`>=`、`<`、`<=` 的边界开闭不同，构造联合键时还要正确填充后续列极值。
- 创建唯一索引前必须扫描已有数据；发现重复键时删除临时索引文件并输出 `failure`。
- 删除并重新创建索引时要清理旧文件描述符对应的缓冲页，避免页号相同造成脏数据复用。
- 后续修改 Insert、Update、Delete 时必须同步维护所有索引。
