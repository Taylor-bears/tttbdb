## 2. 题目要求概述

本题要求在 RMDB 中实现大小为 8 字节的 DATETIME 类型，支持固定格式的时间输入以及在该字段上的增删改查。同时需要严格判断输入合法性，包括格式长度、年份范围、月份、每月天数、闰年和时分秒范围；所有非法输入必须输出 `failure`。

## 4. 实现方法

### 4.1 修改模块

1. 类型系统：增加 DATETIME 字段类型和 AST 类型。
2. Parser：识别 `DATETIME` 关键字并生成 8 字节列定义。
3. Analyze：识别目标 DATETIME 字段，将字符串常量校验并转换。
4. Execution：实现时间记录编码、条件比较和输出格式化。
5. Test：覆盖公开样例、闰年、边界、非法格式和持久化。

### 4.2 实现流程

```text
1. Parser 将 DATETIME 字段记录为 8 字节类型。
2. Analyze 取得 DATETIME 对应的字符串常量。
3. 检查长度、分隔符和数字位置。
4. 解析年月日时分秒并验证实际日历规则。
5. 将合法时间编码为 YYYYMMDDHHMMSS 的 int64_t。
6. Insert/Update 写入 8 字节记录；SeqScan 直接比较编码值。
7. SELECT 将编码值格式化回 YYYY-MM-DD HH:MM:SS。
```

### 4.3 关键伪代码

```text
算法：DATETIME 合法性校验

输入：datetime 字符串 text
输出：64 位时间编码或 failure

1. 若长度不为 19，或固定分隔符不匹配，则失败。
2. 解析 year、month、day、hour、minute、second。
3. 检查 year ∈ [1000, 9999]，month ∈ [1, 12]。
4. 根据月份和闰年计算 max_day，检查 day ∈ [1, max_day]。
5. 检查 hour ≤ 23、minute ≤ 59、second ≤ 59。
6. 按年月日时分秒依次乘 100 并合并为 int64_t。
```

## 7. 调试过程记录

| 问题 | 原因 | 解决方法 |
|---|---|---|
| 直接使用字符串存储会占 19 字节 | 不符合题目规定的 8 字节类型 | 将时间编码为 `YYYYMMDDHHMMSS` 形式的 `int64_t` |
| 只校验日小于等于 31 会接受错误日期 | 不同月份的实际天数不同 | 使用月份天数表并单独处理闰年二月 |
| DATETIME 字面量最初被识别为 STRING | Parser 中时间输入沿用字符串字面量 | Analyze 根据目标列类型完成校验和 DATETIME 转换 |
| 8 字节值可能位于未对齐偏移 | 记录中前置字段长度不固定 | 使用 `memcpy` 读写 64 位编码，避免未对齐访问 |
