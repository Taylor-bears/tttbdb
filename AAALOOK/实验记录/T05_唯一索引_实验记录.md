## 2. 题目要求概述

本题要求 RMDB 支持单列和多列唯一索引，包括索引创建、删除、展示、单点查询和范围查询。查询必须真正使用索引，并按照联合索引最左匹配原则选择扫描范围。基表发生插入、删除和更新时，索引需要同步变化，同时保证索引键唯一。

## 4. 实现方法

### 4.1 修改模块

1. `index`：补全 B+ 树节点操作、插入分裂、边界查找和叶节点扫描。
2. `system`：管理索引文件、索引元数据和 `show index` 输出。
3. `optimizer`：识别可用索引并实现联合索引最左匹配。
4. `execution`：执行单点/范围索引扫描并同步维护索引。
5. `parser`：增加展示索引语句。

### 4.2 实现流程

```text
创建索引：
1. 校验表、索引列及重复索引。
2. 创建并打开索引文件。
3. 扫描基表，将联合键与 RID 插入 B+ 树。
4. 若出现重复键，清理临时索引并返回 failure。
5. 保存索引元数据并刷新数据库元数据。

查询索引：
1. Planner 根据最左匹配选择最佳索引。
2. IndexScan 根据等值/范围条件构造上下界联合键。
3. 通过 B+ 树 lower_bound/upper_bound 定位叶节点槽位。
4. 沿叶节点链表扫描边界范围并读取基表记录。
5. 使用完整 WHERE 条件进行最终过滤。
```

### 4.3 关键伪代码

```text
算法：唯一索引插入

输入：联合键 key、记录位置 rid
输出：成功或 failure

1. 从根节点按关键字定位目标叶节点。
2. 在叶节点二分查找 key；若已存在则返回 failure。
3. 将 (key, rid) 插入有序位置。
4. 若节点达到容量上限，将右半部分移动到新节点。
5. 把新节点首键插入父节点；父节点溢出时递归分裂。
6. 必要时创建新根节点并更新索引文件头。
```

## 7. 调试过程记录

| 问题 | 原因 | 解决方法 |
|---|---|---|
| 删除旧索引后创建新索引出现虚假 RID | 操作系统复用文件描述符，缓冲池仍保留旧索引页 | 关闭索引时刷新并清除该文件描述符的全部缓冲页 |
| 联合索引条件顺序不同无法命中 | 原规则按 WHERE 条件出现顺序拼接字段 | 按索引列顺序查找条件，实现与 SQL 顺序无关的最左匹配 |
| 范围扫描可能包含错误边界 | 严格与非严格比较需要不同起止位置 | 分别使用 lower_bound/upper_bound，并用极值补全联合键 |
| 更新多条记录时可能部分成功后才发现重复 | 逐行修改时唯一性检查不具备语句级一致性 | 先计算所有新键并统一校验，再批量删除旧键、更新记录和插入新键 |
| 对已有数据建索引遇到重复键 | 唯一性只能在装载过程中发现 | 捕获重复异常，关闭并销毁临时索引文件，不写入元数据 |

## 8. 可放入实验报告的总结

本题完成了 RMDB 唯一索引功能。系统以 B+ 树组织索引键和记录 RID，支持节点分裂、单点查找及范围扫描；查询规划器根据最左匹配原则选择联合索引并生成真实的索引扫描计划。索引 DDL、展示、持久化以及增删改同步维护均已实现，所有修改在执行前统一检查唯一性。最终希冀平台一次性通过，本地功能、压力、性能和旧题回归测试也全部通过。
