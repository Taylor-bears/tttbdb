## 2. 题目要求概述

本题要求 RMDB 支持 `COUNT`、`MAX`、`MIN` 和 `SUM` 聚合函数。`COUNT`、`MAX`、`MIN` 需要处理整数、浮点数和字符串字段，`SUM` 处理整数与浮点数字段；查询结果需要正确使用 SQL 中的 `AS` 别名，整数不显示小数，浮点数保留六位小数，并将输出写入数据库目录下的 `output.txt`。

## 4. 实现方法

### 4.1 修改模块

1. `parser`：扩展词法、语法和 AST，使 SELECT 能表达聚合函数、星号与别名。
2. `analyze`：解析聚合列，完成表名推断、列存在性检查和参数类型检查。
3. `optimizer`：新增 AggregatePlan，并将其插入扫描/连接与最终投影之间。
4. `execution`：新增 AggregateExecutor，完成计数、求和、最大值和最小值计算。
5. `portal`：把聚合执行计划转换为可运行的执行器树。

### 4.2 实现流程

```text
1. Parser 识别聚合函数、参数和 AS 别名，生成 AggregateExpr。
2. Analyze 补全参数列所属表，并检查 SUM 只能用于 INT/FLOAT。
3. Planner 先生成原有扫描或连接计划，保证 WHERE 条件正常执行。
4. 在输入计划之上建立 AggregatePlan，再建立结果投影计划。
5. AggregateExecutor 扫描全部输入记录并维护每个聚合项的中间状态。
6. 扫描结束后生成一条结果记录，由原输出模块写入客户端和 output.txt。
```

### 4.3 关键伪代码

```text
算法：无分组聚合

输入：过滤后的记录流、聚合项列表
输出：一条聚合结果记录

1. 为每个聚合项初始化计数、和值或极值状态。
2. 对输入记录逐行处理：
   - COUNT：计数加一；
   - SUM：按 INT/FLOAT 读取并累加；
   - MAX/MIN：按字段类型与当前极值比较并更新。
3. 将计数和求和结果编码到输出记录。
4. 使用 AS 别名构造输出列元数据。
5. 返回唯一一条结果记录；空输入的 COUNT 写入 0。
```

## 7. 调试过程记录

| 问题 | 原因 | 解决方法 |
|---|---|---|
| `COUNT(列)` 语法规则出现移进/归约冲突 | `COUNT` 同时出现在通用聚合函数规则和 `COUNT(*)` 特殊规则中 | 将 `COUNT(列)` 独立成明确规则，消除无效产生式 |
| 聚合结果无法沿用普通 SELECT 输出 | 原 Portal 默认 SELECT 根节点只有 ProjectionPlan | 在扫描计划上增加 AggregatePlan，并在外层保留 ProjectionPlan 以复用列标题和输出逻辑 |
| 聚合别名与源列元数据不一致 | 聚合结果不再属于原始表列 | 为聚合结果构造空表名、以 SQL 别名为列名的新 ColMeta |
| 浮点连续求和可能放大误差 | 直接用 `float` 作为中间累计值精度较低 | 内部使用 `double` 累计，生成结果记录时再转换为 `float` |
